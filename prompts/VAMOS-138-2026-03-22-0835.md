## ERROR 1 — Hydration Error (50+ events, https://90soccer.ftable.co...)
"Hydration failed - the server rendered HTML didn't match"

### Find root cause
```
grep -rn "localStorage\|window\.\|document\.\|navigator\." src/app/page.tsx src/app/layout.tsx | grep -v "//\|typeof window" | head -20
```

Hydration errors happen when server renders different HTML than client.
Common causes:
- `localStorage.getItem()` called during render (always undefined on server)
- `Date.now()` or random values in render
- Browser-only APIs used without `typeof window !== 'undefined'` check

Fix pattern:
```tsx
// WRONG — causes hydration mismatch:
const value = localStorage.getItem('key') // undefined on server

// RIGHT — use useEffect or suppressHydrationWarning:
const [value, setValue] = useState('')
useEffect(() => {
  setValue(localStorage.getItem('key') ?? '')
}, [])
```

Search for all localStorage reads in render (not in useEffect):
```
grep -rn "localStorage\.getItem\|localStorage\.get" src/components/game/HomeScreen.tsx src/app/page.tsx | grep -v "useEffect\|//\|typeof" | head -20
```

Fix each one — wrap in useEffect or add `typeof window !== 'undefined'` guard.

Also check for suppressHydrationWarning on root elements:
```
grep -n "suppressHydrationWarning" src/app/layout.tsx
```
If not present on `<html>` and `<body>`, add it.
